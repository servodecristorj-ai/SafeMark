Section 02 / Solution

Strict 104-key matrix validation.

SafeMark validates against a strict 104-key matrix. Only #F8F8F8 (Esc) to #4B5753 (Num.) are allowed. Anything else aborts with "[MALICIOUS BEHAVIOR DETECTED]". Written in pure C++, zero dependencies.

Implementation
constexpr uint32_t MIN_KEY = 0x4B5753; // Num.
constexpr uint32_t MAX_KEY = 0xF8F8F8; // Esc

bool validate(uint32_t c) {
  if (c < MIN_KEY || c > MAX_KEY)
    return log("[MALICIOUS BEHAVIOR DETECTED]"), false;
  return true;
}

Section 04 / Why I Built This

The parser was cured. Markdown was still exposed.

I reported a global null-byte parser flaw 4 months ago. Meta and WhatsApp patched it. The patch fixed the parser — but yesterday, while testing the fix, I found a new way in: a simple markdown sequence > **:** that lets untrusted input slip through.

“Filtros de IA vigiam palavras. SafeMark vigia o que elas viram depois do markdown.”

The parser was cured. Markdown was still exposed.

That's why I built SafeMark Chroma Firewall v1. Not to format text, but to stop anyone from 'entering' the stream.
